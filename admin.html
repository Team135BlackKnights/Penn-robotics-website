How to Use Image Slots

            
Image Slots let you swap out photos and backgrounds on the live website without touching any code. Here's everything you need to know.


            
    
                
  1. 
                    Find the image you want to change
                    
    Each card in this section represents one image on the site. The card title tells you what it is (e.g. "Landing Hero Background") and a small preview shows the current image.
    
                
    2. 
                
  2. 
                    Upload a new image
                    
    Click Choose File (or drag & drop an image file directly onto the card), then click Upload The new image goes live on the website immediately.
    
                    
    Accepted formats are listed on each card. Keep file sizes reasonable — very large files will slow the page down for visitors.
    
                
    3. 
                
  3. 
                    Preview before you commit
                    
    After selecting a file, a small preview appears on the card so you can check it looks right before hitting Upload.
    
                    
    You can also click any existing preview image to open it full-size in a lightbox.
    
                
    4. 
                
  4. 
                    Undo / Reset to default
                    
    Click the red Reset to default button on a card to remove your uploaded image and go back to the original built-in image. This cannot be undone, so make sure you have a copy if you want to keep your upload.
    
                
    5. 
                
  5. 
                    Finding a specific slot
                    
    Use the Search box to type any part of the slot name or description. Use the Filter button to narrow down by which page the image is on, or whether it has been uploaded or is still the default.
    
                
    6. 
                
  6. 
                    View the image in context
                    
    Each card has a View on site link that opens the relevant page and scrolls directly to where that image appears, so you can see it in context.
    
                
    7. 
            


            

                
                Changes take effect immediately on the live site — no need to refresh or redeploy anything. Just upload and you're done.
            

        

    


    
    

        

            
            
How to Add a Dynamic Image Slot

            
Dynamic image slots let you upload/swap images for any element on any page straight from this admin panel — no code deploys needed.


            
    
                
  1. 
                    Pick a unique key
                    
    Keys follow the pattern pagename.sectionname, e.g. outreach.hero. Keep them lowercase with dots or hyphens only.
    
                    
    What this does: the key is the identifier both the website and the admin panel use to refer to the same image. Think of it as a label the code looks up when it needs to fetch or display that image.
    
                
    2. 
                
  2. 
                    Add the key to api/image_keys.json
                    
    Add one entry inside the top-level JSON object:
    
                    
    "outreach.hero": {
  "label": "Outreach Hero Background",
  "apply": "background",
  "default": "/outreach/media/default.webp",
  "page": "/outreach.html",
  "description": "Hero background for the Outreach page",
  "allowed": ["image/png", "image/jpeg", "image/webp"],
  "max_size": 524288000
}
    
                    
      
                        
    • label — friendly name shown in this dashboard
      • 
                        
    • apply — "background" sets it as a CSS background; "src" sets it on an <img> tag's src
      • 
                        
    • default — fallback path used when no image has been uploaded yet
      • 
                        
    • page — which page this slot lives on (for filtering here in the dashboard)
      • 
                        
    • allowed — MIME types accepted for upload
      • 
                    
    
                    
    What this does: the JSON entry supplies metadata the loader and admin UI use — the loader uses default when no upload exists, and the admin UI uses label, page, and allowed to render the upload card and enforce restrictions.
    
                
    3. 
                
  3. 
                    Add the data-image-slot attribute to the HTML element
                    
    On whichever <div>, <section>, or <img> should show the dynamic image, add:
    
                    
    <div class="landing" data-image-slot="outreach.hero">
    
                    
    What this does: the page element tells image-slot-loader.js which key it needs. On page load the loader scans for elements with data-image-slot, then asks the API for that key's current image and applies it to the element.
    
                
    4. 
                
  4. 
                    Include image-slot-loader.js in the page's <head>
                    
    If it's not already there, add:
    
                    
    <script src="image-slot-loader.js" defer></script>
    
                    
    What this does: the loader runs early (on DOMContentLoaded). It fetches api/image_keys.json for metadata and makes per-slot API calls to /get-image?key=.... When a response contains an uploaded image URL the loader applies it (or it falls back to the default entry).
    
                
    5. 
                
  5. 
                    Remove any hardcoded background-image from the element's SCSS/CSS
                    
    The loader will apply the image dynamically, so any static background-image on that selector will conflict. Comment it out and leave a note:
    
                    
    // background-image removed — managed by image-slot-loader.js via data-image-slot
    
                    
    What this does: if a CSS rule sets a static background, the loader's dynamic image can be hidden underneath or create visual glitches. Removing the static image ensures the loader's image is what visitors see.
    
                
    6. 
                
  6. 
                    Done!
                    
    Reload this dashboard — the new slot will appear here in the Image Slots section. Upload an image and it will go live immediately.
    
                    
    What happens next: the admin UI requests the list of image keys from the server, finds your new entry, and creates a card for it. When you upload an image from that card the admin sends it to the API, which stores it and returns a URL — the site (and the loader) then serve that uploaded image instead of the default.
    
                
    7. 
            


            

                
                The loader starts elements invisible and fades them in once the image loads, so there's never a flash of the default CSS background.
